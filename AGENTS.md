# AGENTS.md

## Repo shape and source of truth
- This repo is a GitHub profile repository (content is the root `README.md`).
- Treat `template/README.md` as the editable source for profile content.
- `README.md` is partially generated by CI: workflow `add-lichess.yml` overwrites it from `template/README.md` and appends a generated Lichess stats block.

## High-impact workflows
- `/.github/workflows/add-lichess.yml`: on push/PR to `main` and monthly schedule, regenerates `README.md` and commits it back to the PR/head branch.
- `/.github/workflows/snake.yml`: on push/PR to `main` and monthly schedule, generates snake assets and publishes them to branch `output`.
- `/.github/workflows/pre-commit.yml`: runs pre-commit checks on PRs and pushes to `main`.
- `/.github/workflows/pre-commit-autoupdate.yml`: weekly updates `.pre-commit-config.yaml` hook versions via an automated PR.

## Local verification commands
- Run full checks with `pre-commit run -a`.
- Auto-fix hooks may rewrite files (notably `codespell` uses `--write-changes`), so re-check git diff after running.

## Editing guidance for agents
- If changing profile sections, edit `template/README.md` first; mirror/update `README.md` only when needed for immediate consistency.
- Do not hand-edit only the Lichess block in `README.md`; CI regenerates that section.
- Keep README links to generated snake assets targeting `output` branch paths (for example `raw.githubusercontent.com/.../output/...`).

## Secrets and CI assumptions
- `add-lichess.yml` expects `API_TOKEN` secret in workflow env.
- `snake.yml` publishes using `${{ secrets.GH_TOKEN }}`.
